### 2.1 致命的問題

| # | 問題 | 詳細 | 影響 |

## 5. ホスティング移行先比較

### 5.1 比較表

| 項目 | GitHub Pages | Netlify | Cloudflare Pages | さくら継続改善 |
|------|-------------|---------|------------------|---------------|
| **月額** | 無料 | 無料（100GB/月） | 無料（無制限帯域） | 約500円/月〜 |
| **SSL自動** | ✅ Let's Encrypt | ✅ 自動 | ✅ 自動 | △ 手動設定要 |
| **git pushデプロイ** | ✅ GitHub Actions | ✅ 自動 | ✅ 自動 | ❌ FTPのみ |
| **Astroビルド** | ✅ Actions対応 | ✅ ネイティブ対応 | ✅ 対応 | ❌ 静的のみ |
| **カスタムドメイン** | ✅ | ✅ | ✅ | ✅ 既存 |
| **CDN** | ✅ Fastly | ✅ グローバル | ✅ 世界最大級 | ❌ 国内1拠点 |
| **ビルド時間制限** | 10分/回 | 300分/月 | 500回/月 | N/A |
| **日本語情報** | ◎ 豊富 | ○ 多い | ○ 増加中 | ◎ 日本企業 |
| **難易度** | 低 | 低 | 低〜中 | 高（改善困難） |

### 5.2 推奨: **Cloudflare Pages**

**理由:**
1. **帯域無制限**（きのこ写真が大量でも安心）
2. **世界最大CDN**（表示速度が最速クラス）
3. **完全無料**（ビルド500回/月は十分すぎる）
4. **git push自動デプロイ**
5. **SSL自動**（現在の証明書問題が即解決）

**次点: Netlify** - 月100GBの帯域制限があるが、このサイト規模なら十分。UIが最も直感的。

### 5.3 ドメイン戦略

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| **A. 独自ドメイン取得** | プロフェッショナル、永続性 | 年間1,000〜2,000円 |
| 例: `saitama-kinoko.org` | | |
| **B. Cloudflareサブドメイン** | 無料、即座に開始 | ブランド力低い |
| 例: `saitama-kinoko.pages.dev` | | |
| **C. さくらサブドメイン維持** | 既存URLの維持 | さくら契約が必要 |

## 6. 技術選定

### 6.1 フレームワーク: **Astro**

| 選定理由 |
|---------|
| コンテンツサイトに最適化されたSSG |
| Markdownで記事管理 → 更新が簡単 |
| Content Collections機能で例会報告・会報を構造化管理 |
| ビルド後は純粋な静的HTML → 高速・安全 |
| Tailwind CSS統合が容易 |

### 6.2 コンテンツ管理方式

```
src/content/
├── reikai/           # 例会報告（Markdown）
│   ├── 2025/
│   │   ├── 04-minoyama.md
│   │   └── 06-chichibu.md
│   └── 2024/
│       └── ...
├── kaihou/           # 会報バックナンバー
│   ├── 035.md
│   └── ...
├── news/             # お知らせ
│   └── 2025-04-update.md
└── data/             # きのこ関連データ
    └── gakumei.json  # 学名データ（JSON化）
```

**記事追加の手順:**
```bash
# 1. Markdownファイルを作成
vim src/content/reikai/2026/04-minoyama.md

# 2. コミット & プッシュ
git add . && git commit -m "docs: 2026年4月美の山報告" && git push

# 3. 自動でビルド＆デプロイ（完了）
```
